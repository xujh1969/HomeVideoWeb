# 家庭影视平台 - 数据库设计文档

## 一、数据库概述

| 属性 | 值 |
|------|-----|
| **数据库类型** | SQLite |

## 二、数据库 Schema

### 2.1 movies（电影表）

存储所有电影信息。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 电影唯一标识 |
| file_path | TEXT | NOT NULL, UNIQUE | 电影文件完整路径 |
| file_size | INTEGER | - | 文件大小（字节） |
| file_mtime | TEXT | - | 文件最后修改时间（ISO8601） |
| ext | TEXT | NOT NULL | 文件扩展名（如.mp4, .mkv） |
| filename_rating | REAL | - | 从文件名解析的评分 |
| filename_genre | TEXT | - | 从文件名解析的分类 |
| title_cn | TEXT | - | 中文标题 |
| title_en | TEXT | - | 英文标题 |
| search_key | TEXT | - | 拼音首字母搜索键（如"wjjh"代表"挽救计划"） |
| tmdb_id | INTEGER | - | TMDB电影ID |
| overview | TEXT | - | 剧情简介 |
| release_date | TEXT | - | 上映日期 |
| runtime | INTEGER | - | 影片时长（分钟） |
| director | TEXT | - | 导演（逗号分隔） |
| cast | TEXT | - | 演员列表（逗号分隔） |
| poster_path | TEXT | - | TMDB海报路径 |
| local_poster | TEXT | - | 本地海报路径（如/api/posters/movie_123.jpg） |
| genres | TEXT | - | 电影类型（逗号分隔） |
| tmdb_rating | REAL | - | TMDB评分 |
| imdb_rating | REAL | - | IMDb评分 |
| douban_rating | REAL | - | 豆瓣评分 |
| vote_count | INTEGER | - | 投票数量 |
| metadata_status | TEXT | DEFAULT 'pending' | 元数据状态 |
| metadata_updated | TEXT | - | 元数据最后更新时间 |
| batch_id | INTEGER | DEFAULT 0 | 批次ID（用于增量更新） |
| created_at | TEXT | DEFAULT (datetime('now')) | 记录创建时间 |
| updated_at | TEXT | DEFAULT (datetime('now')) | 记录更新时间 |

**索引**:
```sql
CREATE INDEX idx_movies_genre ON movies(filename_genre);
CREATE INDEX idx_movies_rating ON movies(filename_rating DESC);
CREATE INDEX idx_movies_title ON movies(title_cn);
CREATE INDEX idx_movies_search_key ON movies(search_key);
```

| title_en | TEXT | - | 英文标题 |

| first_air_date | TEXT | - | 首播日期 |

### 2.3 episodes（剧集集数表）

存储剧集的每一集信息。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 集数唯一标识 |
| series_id | INTEGER | NOT NULL, FK | 所属剧集ID |
| file_path | TEXT | NOT NULL, UNIQUE | 集文件完整路径 |
| file_size | INTEGER | - | 文件大小（字节） |
| file_mtime | TEXT | - | 文件修改时间 |
| ext | TEXT | NOT NULL | 文件扩展名 |
| season_number | INTEGER | NOT NULL, DEFAULT 1 | 季号（第几季） |
| episode_number | INTEGER | NOT NULL | 集号（第几集） |
| episode_title | TEXT | - | 集标题 |
| watched | INTEGER | DEFAULT 0 | 是否已看（0=未看，1=已看） |
| progress | REAL | DEFAULT 0 | 播放进度（0-1之间的小数） |
| batch_id | INTEGER | DEFAULT 0 | 批次ID |
| created_at | TEXT | DEFAULT (datetime('now')) | 创建时间 |
| updated_at | TEXT | DEFAULT (datetime('now')) | 更新时间 |

**外键约束**:
```sql
FOREIGN KEY (series_id) REFERENCES series(id) ON DELETE CASCADE
```

**索引**:
```sql
CREATE INDEX idx_episodes_series ON episodes(series_id, season_number, episode_number);
CREATE INDEX idx_episodes_file ON episodes(file_path);
```

### 2.4 sources（媒体源表）

存储配置的媒体源（电影/剧集存放位置）。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 源唯一标识 |
| name | TEXT | NOT NULL | 源名称（显示用） |
| source_type | TEXT | NOT NULL, DEFAULT 'smb' | 源类型（smb/local/nfs） |
| address | TEXT | NOT NULL | 地址（IP/路径） |
| port | INTEGER | - | 端口（SMB用） |
| username | TEXT | - | 用户名（SMB用） |
| password | TEXT | - | 密码（SMB用，已加密存储） |
| directory | TEXT | NOT NULL | 共享目录 |
| mount_point | TEXT | - | 挂载点路径 |
| enabled | INTEGER | DEFAULT 1 | 是否启用（0=禁用，1=启用） |
| scan_interval | INTEGER | DEFAULT 3600 | 扫描间隔（秒，默认1小时） |
| last_scan_at | TEXT | - | 最后扫描时间 |
| status | TEXT | DEFAULT 'unknown' | 状态（online/offline/error/unknown） |
| error_message | TEXT | - | 错误信息 |
| created_at | TEXT | DEFAULT (datetime('now')) | 创建时间 |
| updated_at | TEXT | DEFAULT (datetime('now')) | 更新时间 |

**source_type 可选值**:
- `smb`: SMB/CIFS共享（Windows网络邻居）
- `local`: 本地路径
- `nfs`: NFS网络文件系统

**status 可选值**:
- `online`: 在线正常
- `offline`: 离线
- `error`: 错误
- `unknown`: 未知/未检测

### 2.5 watch_history（观看历史表）

存储用户播放历史记录。

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|----------|------|------|
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 历史记录唯一标识 |
| media_type | TEXT | NOT NULL | 媒体类型（movie/episode） |
| media_id | INTEGER | NOT NULL | 媒体ID（电影ID或集数ID） |
| series_id | INTEGER | - | 所属剧集ID（仅剧集类型时使用） |
| watched_at | TEXT | DEFAULT (datetime('now')) | 观看时间 |
| progress | REAL | DEFAULT 0 | 播放进度（0-1） |
| created_at | TEXT | DEFAULT (datetime('now')) | 创建时间 |

**索引**:
```sql
CREATE INDEX idx_watch_history_media ON watch_history(media_type, media_id);
CREATE INDEX idx_watch_history_time ON watch_history(watched_at DESC);
```

## 四、关键设计说明

### 4.1 批次扫描机制（batch_id）

- 每次执行"同步媒体库"时，会生成一个新的 `batch_id`
- `batch_id = 0`: 表示手动添加或迁移的数据，不会被自动清理
- 扫描完成后，系统会比较当前扫描到的文件和数据库记录，删除不再存在的文件记录
- 这种机制保证了增量更新和垃圾清理的自动化

### 4.2 元数据状态（metadata_status）

所有电影和剧集都有元数据状态字段：

| 状态值 | 说明 |
|--------|------|
| `pending` | 待获取，需要从API刮削 |
| `fetched` | 已成功获取元数据 |
| `failed` | 获取失败 |

### 4.3 拼音搜索键（search_key）

`search_key` 字段存储中文标题的拼音首字母，用于快速搜索：

| 电影标题 | search_key 示例 |
|----------|-----------------|
| 挽救计划 | wjjh |
| 利刃出鞘 | lrcq |
| 最后的舞动 | zhdwd |

搜索时，API会同时匹配 `title_cn`、`title_en`、`search_key` 三个字段。

### 4.4 多源评分系统

系统支持三个评分来源：

| 评分字段 | 来源 | 用途 |
|----------|------|------|
| tmdb_rating | TMDB | 国际评分 |
| imdb_rating | IMDb | 国际评分 |
| douban_rating | 豆瓣 | 中文评分 |
| filename_rating | 文件名解析 | 文件名中提取的评分 |

前端展示时会优先使用 douban_rating > imdb_rating > filename_rating。

### 4.5 多态关联设计

`watch_history` 表采用多态关联设计：

- `media_type = 'movie'`: media_id 指向 movies.id
- `media_type = 'episode'`: media_id 指向 episodes.id，series_id 指向 episodes.series_id

这种设计简化了观看历史的管理，无需分别为电影和剧集创建不同的历史表。

### 4.6 剧集结构

剧集数据分为两级：
1. `series` 表：存储剧集基本信息（一行记录代表一个剧集）
2. `episodes` 表：存储每一集信息（多行记录）

剧集详情接口会返回合并后的结构：
```json
{
  "id": 48,
  "title_cn": "最后的舞动",
  "seasons": [
    {
      "season_number": 1,
      "episodes": [
        { "id": 396, "episode_number": 1, ... },
        { "id": 397, "episode_number": 2, ... }
      ]
    }
  ]
}
```

## 七、常见查询示例

### 7.1 获取电影列表（按评分排序）
```sql
SELECT * FROM movies ORDER BY filename_rating DESC LIMIT 24 OFFSET 0;
```

### 7.2 搜索电影（支持拼音）
```sql
SELECT * FROM movies 
WHERE title_cn LIKE '%挽救%' 
   OR title_en LIKE '%挽救%' 
   OR search_key LIKE '%wjjh%';
```

### 7.3 获取剧集详情（含所有集）
```sql
-- 先获取剧集信息
SELECT * FROM series WHERE id = 48;

-- 再获取所有集
SELECT * FROM episodes WHERE series_id = 48 ORDER BY season_number, episode_number;
```

### 7.4 获取最近播放
```sql
SELECT * FROM watch_history ORDER BY watched_at DESC LIMIT 10;
```

### 7.5 按分类统计
```sql
SELECT filename_genre, COUNT(*) as count FROM movies GROUP BY filename_genre;
```
