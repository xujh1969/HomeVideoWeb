## 1. 项目概述

### 1.1 内容分类

本平台管理两类影视内容：

| 类型 | 存储方式 | 命名示例 | 识别依据 |
|------|---------|---------|---------|
| **电影** | 单文件 | `【8.4】【剧情】肖申克的救赎.The Shawshank Redemption.mkv` | 文件直接位于扫描根目录下 |
| **连续剧** | 目录 | `【9.1】【纪录片】夜色中的地球.S02/` → 内含 `S02E01.mkv` … `S02E09.mkv` | 目录内包含多个视频文件 |

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| **视频轮播区** | 首页顶部视频轮播，展示最近添加的5部影片的随机片段（10秒切换），左下角显示影片名称/类型/评分/简介，点击进入详情页。无影片时显示欢迎信息 |
| **最近播放区** | 轮播区下方横向列表，最多展示7部最近观看过的影片，支持快速续播 |
| **影片库分页** | 横向列表下方网格展示影片，每页24部，支持分页导航 |
| **左侧分类导航** | 固定左侧边栏，包含首页/电影库/连续剧主导航按钮，以及电影分类标签（剧情/动作/科幻/纪录片等），点击过滤 |
| **首页模式** | 视频轮播 + 最近播放 + 影片库分页，分类栏显示所有电影的分类和数量 |
| **电影库模式** | 只显示所有电影分页，分类栏自动切换为"全部" |
| **连续剧模式** | 只显示所有连续剧分页，分类栏只显示"全部"按钮 |
| **电影详情页** | 左侧视频播放区（自动播放）+ 右侧简介区（海报、中英文名、导演、演员、剧情简介、评分等），支持双击/按钮全屏，ESC键返回 |
| **连续剧详情页** | 封面海报、简介、分季展示、每集列表（编号+标题），点击单集播放 |
| **网页播放器** | 浏览器内播放，支持全屏、进度拖拽、字幕切换，支持mkv/mp4/webm/avi/mov格式 |
| **设置对话框** | 右上角配置按钮，弹出对话框管理电影源（SMB / 本地路径），配置地址、账号、密码、目录 |
| **电影库刷新** | 手动触发按钮，全量重新扫描所有电影源，自动清理已删除/不可达的文件记录 |
| **多源存储** | 支持 SMB 共享（\\192.168.1.100\video）和本地硬盘扫描 |
| **自动刮削** | 从文件名提取信息后，自动查询 TMDB / OMDb / 豆瓣获取完整元数据 |
| **分页展示** | 影片列表支持分页显示，每页24部，避免一次性加载过多数据 |

### 1.3 非功能需求

- **深色主题**：全局深色系 UI，参考腾讯视频 / 爱奇艺设计风格，减少观影环境光干扰
- **排版布局**：左侧固定导航栏 + 右侧内容区，Hero 大屏沉浸式体验，滚动列表区清晰分层
- **内网部署**：服务仅在内网运行，无需认证/鉴权
- **按需转码**：只在浏览器不支持原始编码时才触发 FFmpeg 转码
- **离线可用**：元数据缓存到本地 SQLite，无网络时仍可浏览
- **响应式**：桌面端优先，兼顾平板访问
- **自动识别**：扫描时自动区分「单文件 = 电影」vs「含多视频的目录 = 连续剧」
- **容错扫描**：库刷新时自动检测文件是否存在，清理失效记录（文件被删除/路径不可达）

## 2. 技术选型总览

| 模块 | 选型 | 理由 |
|------|------|------|
| 语言 | **TypeScript** | 前后端统一，类型安全，接口契约可复用 |
| 后端框架 | **Node.js + Express (TS)** | 与用户现有技术栈一致，社区成熟，TypeScript 类型保障 |
| 前端框架 | **React 18 + Vite + TypeScript** | 组件化开发，TS 严格模式，Vite HMR 极快 |
| 前端路由 | **React Router v6** | 标配，支持嵌套路由和数据加载 |
| UI 框架 | **Tailwind CSS** | 原子化 CSS，设计灵活，无需写样式文件 |
| 代码规范 | **ESLint + Prettier** | 统一代码风格，pre-commit hooks |
| 视频播放器 | **ZWPlayer (React 组件)** | 全协议统一内核，原生支持 H.265/AV1，本地文件直读，React 官方组件，无需插件 |
| 视频转码 | **FFmpeg** | 业界标准，命令行即可调用，支持所有格式 |
| 影视元数据 | **豆瓣 API v2（首选）+ TMDB API v3（增强）** | 豆瓣：国内直连、中文数据全、无需注册(公开key)；TMDB：海报资源全、演职员详细，需hosts修复后可注册 |
| IMDB 评分 | **OMDb API** | 每天 1000 次免费，直接返回 IMDB 评分 |
| 数据库 | **SQLite + better-sqlite3** | 零配置、单文件、性能足以支撑家庭规模，better-sqlite3 同步 API 开发体验好 |
| 文件扫描 | **chokidar** | 监听目录变更，自动发现新增/删除的影片 |
| 文件名解析 | **自定义正则模块** | 用户命名规则规范，正则即可精确提取，无需 guessit 等第三方库 |
| HTTP 服务器 | **express.static** 或 **nginx** | 静态文件 serve，视频直链可走 nginx 以提升性能 |

```sql
-- ========================================
-- 电影表 (单文件影片)
-- ========================================
CREATE TABLE movies (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    file_path       TEXT NOT NULL UNIQUE,      -- 完整文件路径
    file_size       INTEGER,                   -- 文件大小 (bytes)
    file_mtime      TEXT,                      -- 文件修改时间
    ext             TEXT NOT NULL,             -- 扩展名 .mkv .mp4 .avi

    -- 来自文件名解析
    filename_rating   REAL,                    -- 文件名中的评分
    filename_genre    TEXT,                    -- 文件名中的分类
    title_cn          TEXT,                    -- 中文名
    title_en          TEXT,                    -- 英文名

    -- 来自 TMDB 刮削
    tmdb_id           INTEGER,                -- TMDB 影片 ID
    overview          TEXT,                    -- 故事梗概
    release_date      TEXT,                    -- 发行日期
    runtime           INTEGER,                -- 片长 (分钟)
    director          TEXT,                    -- 导演
    cast              TEXT,                    -- 演员 (逗号分隔字符串)
    poster_path       TEXT,                    -- TMDB 海报远程路径
    local_poster      TEXT,                    -- 本地海报缓存路径
    genres            TEXT,                    -- 分类 (逗号分隔字符串)

    -- 评分 (刮削获取)
    tmdb_rating       REAL,                   -- TMDB 评分
    imdb_rating       REAL,                   -- IMDB 评分 (OMDb)
    douban_rating     REAL,                   -- 豆瓣评分
    vote_count        TEXT,                   -- 评分数 (字符串类型)

    -- 状态标记
    metadata_status   TEXT DEFAULT 'pending',  -- pending / fetched / failed
    metadata_updated  TEXT,                    -- 最后刮削时间

    created_at        TEXT DEFAULT (datetime('now')),
    updated_at        TEXT DEFAULT (datetime('now'))
);

-- 索引
CREATE INDEX idx_movies_genre ON movies(filename_genre);
CREATE INDEX idx_movies_rating ON movies(filename_rating DESC);
CREATE INDEX idx_movies_title ON movies(title_cn);


-- ========================================
-- 连续剧表 (目录级)
-- ========================================
CREATE TABLE series (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    dir_path        TEXT NOT NULL UNIQUE,      -- 目录完整路径
    season_count    INTEGER DEFAULT 1,         -- 总季数 (从目录名解析)
    episode_count   INTEGER DEFAULT 0,          -- 总集数 (扫描统计)

    -- 来自目录名解析 (同电影命名规范，多一个 season 字段)
    filename_rating   REAL,
    filename_genre    TEXT,
    title_cn          TEXT,
    title_en          TEXT,
    season_label      TEXT,                    -- 如 "S02"、"Season 2"

    -- 来自 TMDB 刮削 (TV 节目用)

    overview          TEXT,
    first_air_date    TEXT,                    -- 首播日期
    director          TEXT,                    -- 导演
    poster_path       TEXT,
    local_poster      TEXT,
    genres            TEXT,                    -- 逗号分隔的分类字符串

    -- 评分
    tmdb_rating       REAL,
    imdb_rating       REAL,
    douban_rating     REAL,
    vote_count        TEXT,                    -- 评分数 (字符串类型)

    -- 状态
    metadata_status   TEXT DEFAULT 'pending',
    metadata_updated  TEXT,

    created_at        TEXT DEFAULT (datetime('now')),
    updated_at        TEXT DEFAULT (datetime('now'))
);

CREATE INDEX idx_series_genre ON series(filename_genre);
CREATE INDEX idx_series_rating ON series(filename_rating DESC);
CREATE INDEX idx_series_title ON series(title_cn);


-- ========================================
-- 剧集表 (单集文件)
-- ========================================
CREATE TABLE episodes (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    series_id       INTEGER NOT NULL,         -- 所属连续剧
    file_path       TEXT NOT NULL UNIQUE,      -- 单集文件完整路径
    file_size       INTEGER,
    file_mtime      TEXT,
    ext             TEXT NOT NULL,

    -- 剧集编号
    season_number   INTEGER NOT NULL DEFAULT 1,-- 第几季
    episode_number  INTEGER NOT NULL,          -- 第几集
    episode_title   TEXT,                      -- 分集标题 (TMDB 刮削)

    -- 播放状态
    watched         INTEGER DEFAULT 0,         -- 是否已看
    progress        REAL DEFAULT 0,            -- 播放进度 (0~1)

    created_at      TEXT DEFAULT (datetime('now')),
    updated_at      TEXT DEFAULT (datetime('now')),

    FOREIGN KEY (series_id) REFERENCES series(id) ON DELETE CASCADE
);

CREATE INDEX idx_episodes_series ON episodes(series_id, season_number, episode_number);
CREATE INDEX idx_episodes_file ON episodes(file_path);


-- ========================================
-- 电影源配置表 (存储SMB/NFS/本地路径)
-- ========================================
CREATE TABLE sources (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    name            TEXT NOT NULL,             -- 源名称 (如 "NFS-客厅NAS")
    source_type     TEXT NOT NULL DEFAULT 'smb',-- 'smb' | 'local' | 'nfs'
    address         TEXT NOT NULL,             -- SMB地址 \\192.168.1.100\video 或盘符 D:\
    port            INTEGER,                   -- 端口 (SMB默认445)
    username        TEXT,                      -- SMB用户名 (local类型为空)
    password        TEXT,                      -- SMB密码 (加密存储)
    directory       TEXT NOT NULL,             -- 扫描目录相对路径 (如 "Movies" 或 "")
    mount_point     TEXT,                      -- 本地挂载点 (如 "Z:" 或 "/mnt/video")
    enabled         INTEGER DEFAULT 1,         -- 是否启用
    scan_interval   INTEGER DEFAULT 3600,      -- 自动扫描间隔 (秒, 0=仅手动)
    last_scan_at    TEXT,                      -- 上次扫描时间
    status          TEXT DEFAULT 'unknown',    -- 'online' | 'offline' | 'error' | 'unknown'
    error_message   TEXT,                      -- 最近错误信息

    created_at      TEXT DEFAULT (datetime('now')),
    updated_at      TEXT DEFAULT (datetime('now'))
);


-- ========================================
-- 观看历史表
-- ========================================
CREATE TABLE watch_history (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    media_type      TEXT NOT NULL,             -- 'movie' | 'episode'
    media_id        INTEGER NOT NULL,          -- movies.id 或 episodes.id
    series_id       INTEGER,                   -- 如果是剧集, 关联series.id (方便查询)
    watched_at      TEXT DEFAULT (datetime('now')),
    progress        REAL DEFAULT 0,            -- 播放进度 (0~1)

    created_at      TEXT DEFAULT (datetime('now'))
);

CREATE INDEX idx_watch_history_media ON watch_history(media_type, media_id);
CREATE INDEX idx_watch_history_time ON watch_history(watched_at DESC);
```

### 4.2 扫描识别逻辑

```
扫描根目录 (如 Z:\video)
  │
  ├── 遍历子目录
  │     │
  │     ├── 目录内包含 ≥2 个视频文件 (.mkv/.mp4/...)
  │     │   → 识别为「连续剧」
  │     │   → 解析目录名 → INSERT/UPDATE series 表
  │     │   → 遍历目录内各视频文件 → INSERT/UPDATE episodes 表
  │     │
  │     └── 目录内只有 1 个视频文件
  │         → 视为单文件电影 (非典型，但兼容)
  │
  └── 遍历根目录下的视频文件
        → 识别为「电影」
        → 解析文件名 → INSERT/UPDATE movies 表
```

### 4.3 数据流

```
  sources 表 (配置)
       │
       ├── SMB: 通过 mount_point (Z:/Y:) 访问
       ├── 本地: 直接路径 (D:\Movies\)
       └── NFS: 挂载点 (/mnt/video/)
       │
       ▼
  libraryRefresh 服务 — 遍历所有 enabled 的 source
       │
       ├── 单视频文件 → 文件名解析器 → movies 表
       │
       └── 目录含多视频 → 目录名解析器 → series 表
                              │
                              └── 逐文件解析集号 → episodes 表
       │
       ├── 清理: DELETE records WHERE file_path NOT EXISTS in filesystem
       │
       ▼ (metadata_status = 'pending')
  元数据刮削服务 → 豆瓣API(首选) → TMDB(增强) → OMDb(IMDB)
       │            (电影走 /movie/:id, 连续剧走 /tv/:id)
       ▼
  UPDATE 详情 + 下载海报
       │ (metadata_status = 'fetched')
       ▼
  React 前端 → API 查询 → 展示
       │
       ├── 播放 → watch_history 记录
       └── 设置 → sources 表 CRUD
```

#### 库刷新流程 (含删除处理)

```
1. 遍历所有 enabled sources
     │
2. 对每个 source，递归扫描目录
     ├── 发现新文件 → INSERT
     ├── 发现已存在文件 → UPDATE file_mtime/file_size
     └── 记录所有存在的 file_path 到内存集合 existingPaths
     │
3. 清理阶段 (扫描全部完成后)
     ├── SELECT file_path FROM movies
     │     WHERE file_path NOT IN (existingPaths)
     │   → DELETE 这些记录 (文件已被删除/移动)
     │
     ├── SELECT file_path FROM episodes
     │     WHERE file_path NOT IN (existingPaths)
     │   → DELETE 这些记录
     │
     └── DELETE FROM series WHERE id NOT IN
           (SELECT DISTINCT series_id FROM episodes)
         (清理没有剧集的空连续剧记录)
     │
4. 更新 sources.last_scan_at
```

```typescript
// server/services/sourceManager.ts

import { db } from '../db'
import type { MovieSource, SourceInput } from '../../shared/types'
import { execSync } from 'child_process'
import fs from 'fs'
import path from 'path'

/** 获取所有源 */
export function getAllSources(): MovieSource[] {
  return db.prepare('SELECT * FROM sources ORDER BY id').all() as MovieSource[]
}

/** 添加源 */
export function addSource(input: SourceInput): number {
  const stmt = db.prepare(`
    INSERT INTO sources (name, source_type, address, port, username, password, directory, mount_point, enabled, scan_interval)
    VALUES (@name, @source_type, @address, @port, @username, @password, @directory, @mount_point, @enabled, @scan_interval)
  `)
  const result = stmt.run({
    name: input.name,
    source_type: input.source_type,
    address: input.address,
    port: input.port ?? null,
    username: input.username ?? null,
    password: input.password ?? null,
    directory: input.directory,
    mount_point: input.mount_point ?? null,
    enabled: input.enabled !== false ? 1 : 0,
    scan_interval: input.scan_interval ?? 3600,
  })
  return result.lastInsertRowid as number
}

/** 更新源 */
export function updateSource(id: number, input: Partial<SourceInput>): void {
  const fields: string[] = []
  const params: Record<string, unknown> = { id }

  for (const [key, value] of Object.entries(input)) {
    if (value !== undefined) {
      fields.push(`${key} = @${key}`)
      params[key] = value
    }
  }
  fields.push("updated_at = datetime('now')")

  db.prepare(`UPDATE sources SET ${fields.join(', ')} WHERE id = @id`).run(params)
}

/** 删除源 */
export function deleteSource(id: number): void {
  db.prepare('DELETE FROM sources WHERE id = ?').run(id)
}

/** 测试源连通性 */
export function testSourceConnection(source: MovieSource): { status: string; message: string } {
  try {
    if (source.source_type === 'local') {
      // 本地路径: 直接检查目录是否存在
      const testPath = path.join(source.address, source.directory)
      if (fs.existsSync(testPath)) {
        const files = fs.readdirSync(testPath).filter(f =>
          ['.mkv', '.mp4', '.avi', '.webm', '.ts', '.mov'].includes(path.extname(f).toLowerCase())
        )
        return { status: 'online', message: `连接成功, 发现 ${files.length} 个视频文件` }
      }
      return { status: 'offline', message: `目录不存在: ${testPath}` }
    }

    if (source.source_type === 'smb') {
      // SMB: 尝试访问路径
      const smbPath = source.mount_point
        ? path.join(source.mount_point, source.directory)
        : `\\\\${source.address}\\${source.directory}`

      if (fs.existsSync(smbPath)) {
        const files = fs.readdirSync(smbPath).filter(f =>
          ['.mkv', '.mp4', '.avi', '.webm', '.ts', '.mov'].includes(path.extname(f).toLowerCase())
        )
        return { status: 'online', message: `连接成功, 发现 ${files.length} 个视频文件` }
      }
      return { status: 'offline', message: `无法访问路径: ${smbPath}` }
    }

    return { status: 'unknown', message: `不支持的源类型: ${source.source_type}` }
  } catch (err: any) {
    return { status: 'error', message: `连接失败: ${err.message}` }
  }
}

/** 构建源的完整扫描路径 */
export function buildScanPath(source: MovieSource): string {
  if (source.source_type === 'local') {
    return path.join(source.address, source.directory)
  }
  if (source.mount_point) {
    return path.join(source.mount_point, source.directory)
  }
  return `\\\\${source.address}\\${source.directory}`
}
```

### 5.4 后端服务 — 库刷新

```typescript
// server/services/libraryRefresh.ts

import { getAllSources, buildScanPath } from './sourceManager'
import { scanDirectory } from './fileScanner'
import { db } from '../db'
import fs from 'fs'
import path from 'path'

interface RefreshProgress {
  status: 'idle' | 'scanning' | 'cleaning' | 'done' | 'error'
  current_source: string
  found_movies: number
  found_series: number
  found_episodes: number
  removed_movies: number
  removed_episodes: number
  errors: string[]
}

let currentProgress: RefreshProgress = {
  status: 'idle',
  current_source: '',
  found_movies: 0,
  found_series: 0,
  found_episodes: 0,
  removed_movies: 0,
  removed_episodes: 0,
  errors: [],
}

export function getRefreshStatus(): RefreshProgress {
  return { ...currentProgress }
}

export async function refreshLibrary(): Promise<void> {
  if (currentProgress.status === 'scanning' || currentProgress.status === 'cleaning') {
    throw new Error('刷新任务正在进行中')
  }

  currentProgress = {
    status: 'scanning',
    current_source: '',
    found_movies: 0,
    found_series: 0,
    found_episodes: 0,
    removed_movies: 0,
    removed_episodes: 0,
    errors: [],
  }

  const allExistingPaths = new Set<string>()

  try {
    const sources = getAllSources().filter(s => s.enabled)

    for (const source of sources) {
      currentProgress.current_source = source.name
      try {
        const scanPath = buildScanPath(source)
        if (!fs.existsSync(scanPath)) {
          currentProgress.errors.push(`${source.name}: 路径不可达 ${scanPath}`)
          continue
        }

        const result = scanDirectory(scanPath, allExistingPaths)
        currentProgress.found_movies += result.movies
        currentProgress.found_series += result.series
        currentProgress.found_episodes += result.episodes
      } catch (err: any) {
        currentProgress.errors.push(`${source.name}: ${err.message}`)
      }
    }

    // ====== 清理阶段 ======
    currentProgress.status = 'cleaning'

    // 1. 找出数据库中但不在文件系统中的电影
    const dbMovies = db.prepare('SELECT id, file_path FROM movies').all() as any[]
    for (const movie of dbMovies) {
      if (!allExistingPaths.has(movie.file_path)) {
        db.prepare('DELETE FROM movies WHERE id = ?').run(movie.id)
        currentProgress.removed_movies++
      }
    }

    // 2. 找出被删除的剧集文件
    const dbEpisodes = db.prepare('SELECT id, file_path FROM episodes').all() as any[]
    for (const ep of dbEpisodes) {
      if (!allExistingPaths.has(ep.file_path)) {
        db.prepare('DELETE FROM episodes WHERE id = ?').run(ep.id)
        currentProgress.removed_episodes++
      }
    }

    // 3. 清理空连续剧 (没有episodes的series)
    db.prepare(`
      DELETE FROM series WHERE id NOT IN (
        SELECT DISTINCT series_id FROM episodes
      )
    `).run()

    // 4. 更新源的扫描时间
    db.prepare("UPDATE sources SET last_scan_at = datetime('now'), status = 'online'").run()

    currentProgress.status = 'done'
  } catch (err: any) {
    currentProgress.status = 'error'
    currentProgress.errors.push(`刷新失败: ${err.message}`)
  }
}
```

```typescript
import path from 'node:path'
import type { ParsedFilename } from '../../shared/types'

export function parseMovieFilename(filePath: string): ParsedFilename {
  const ext = path.extname(filePath).toLowerCase()
  const stem = path.basename(filePath, ext)

  // 提取所有【】标签
  const tagRegex = /【([^】]+)】/g
  const tags: string[] = []
  let match: RegExpExecArray | null
  while ((match = tagRegex.exec(stem)) !== null) {
    tags.push(match[1])
  }

  // 区分评分和分类
  let rating: number | null = null
  let genre: string | null = null
  for (const tag of tags) {
    if (/^\d+(\.\d+)?$/.test(tag)) {
      rating = parseFloat(tag)
    } else {
      genre = tag
    }
  }

  // 去掉所有标签，剩余部分按 . 分割
  const rest = stem.replace(/【[^】]+】/g, '').trim()
  const parts = rest.split('.')

  const isChinese = (s: string): boolean => /[\u4e00-\u9fff]/.test(s)
  const titleCN = parts.filter(isChinese).join('.') || null
  const titleEN = parts
    .filter(s => !isChinese(s) && s.trim())
    .join(' ') || null

  return { rating, genre, titleCN, titleEN, ext }
}
```

### 7.5 连续剧目录名解析 (TypeScript)

```typescript
import path from 'node:path'
import type { ParsedDirName } from '../../shared/types'

const SEASON_PATTERNS = [
  /\.S(\d{1,2})$/i,                  // .S02, .s1
  /\.Season\s*(\d{1,2})$/i,         // .Season 2, .season1
  /第(\d{1,2})季$/,                   // 第2季 (中文)
]

export function parseSeriesDirName(dirPath: string): ParsedDirName {
  const dirName = path.basename(dirPath)

  // 提取所有【】标签
  const tagRegex = /【([^】]+)】/g
  const tags: string[] = []
  let match: RegExpExecArray | null
  while ((match = tagRegex.exec(dirName)) !== null) {
    tags.push(match[1])
  }

  let rating: number | null = null
  let genre: string | null = null
  for (const tag of tags) {
    if (/^\d+(\.\d+)?$/.test(tag)) {
      rating = parseFloat(tag)
    } else {
      genre = tag
    }
  }

  // 去掉标签
  let rest = dirName.replace(/【[^】]+】/g, '').trim()

  // 提取季节号
  let seasonNumber = 1
  let seasonLabel: string | null = null
  for (const pattern of SEASON_PATTERNS) {
    const m = rest.match(pattern)
    if (m) {
      seasonNumber = parseInt(m[1], 10)
      seasonLabel = m[0].replace(/^\./, '')  // 去前导点: "S02"
      rest = rest.replace(pattern, '').trim()
      break
    }
  }

  // 剩余部分按 . 分割
  const parts = rest.split('.')

  const isChinese = (s: string): boolean => /[\u4e00-\u9fff]/.test(s)
  const titleCN = parts.filter(isChinese).join('.') || null
  const titleEN = parts
    .filter(s => !isChinese(s) && s.trim())
    .join(' ') || null

  return { rating, genre, titleCN, titleEN, seasonLabel, seasonNumber }
}
```

### 7.6 剧集文件名解析 (TypeScript)

```typescript
import path from 'node:path'
import type { ParsedEpisodeName } from '../../shared/types'

/**
 * 解析单集文件名，返回季号和集号
 * @param filePath  单集文件路径
 * @param fallbackSeason 从目录名解析出的默认季号
 */
export function parseEpisodeFilename(
  filePath: string,
  fallbackSeason: number
): ParsedEpisodeName {
  const ext = path.extname(filePath).toLowerCase()
  const stem = path.basename(filePath, ext)

  let seasonNumber = fallbackSeason
  let episodeNumber = 1

  // 模式1: S02E05 → season=2, episode=5
  const seMatch = stem.match(/S(\d{1,2})\s*E(\d{1,3})/i)
  if (seMatch) {
    seasonNumber = parseInt(seMatch[1], 10)
    episodeNumber = parseInt(seMatch[2], 10)
    return { seasonNumber, episodeNumber, ext }
  }

  // 模式2: E05 → episode=5, season 用 fallback
  const eMatch = stem.match(/E(\d{1,3})/i)
  if (eMatch) {
    episodeNumber = parseInt(eMatch[1], 10)
    return { seasonNumber, episodeNumber, ext }
  }

  // 模式3: 纯数字 (02, 03, 12...) → episode=数字
  const numMatch = stem.match(/(\d{1,3})$/)
  if (numMatch) {
    const num = parseInt(numMatch[1], 10)
    // 如果数字 ≥ 100，可能是 S01E01 被压缩的变体（极少见，保守处理）
    if (num < 100) {
      episodeNumber = num
      return { seasonNumber, episodeNumber, ext }
    }
  }

  // 回退：无法解析时设为第1集
  return { seasonNumber: fallbackSeason, episodeNumber: 1, ext }
}
```

#### 8.3.2 在 metadataFetcher.ts 中被调用

OMDb 是刮削的**最后一步**——豆瓣和 TMDB 先拿到电影基本信息后，用 IMDB ID 补上评分。

```typescript

import { fetchMovieByDouban, searchDouban } from './doubanService'
import { fetchMovieByTmdbId } from './tmdbService'
import { fetchMovieByImdbId } from './omdbService'

export async function fetchMovieMetadata(titleCN: string, titleEN?: string) {
  // 1️⃣ 豆瓣：搜索 + 获取详情（中文元数据 + 评分）
  const doubanMovie = await searchDouban(titleCN || titleEN)
  if (!doubanMovie) return null    // 什么都没搜到，标记 failed

  const detail = await fetchMovieByDouban(doubanMovie.id)

  // 2️⃣ TMDB（可选，增强海报和英文信息）

    omdbData = await fetchMovieByImdbId(imdbId)       // ← 调用 OMDb
  }

  // 4️⃣ 合并写入 SQLite
  return {
    titleCN:       detail?.title || doubanMovie.title,
    titleEN:       detail?.original_title || titleEN,
    summary:       detail?.summary,
    directors:     detail?.directors?.map(d => d.name),
    actors:        detail?.casts?.slice(0, 8)?.map(c => c.name),
    genres:        detail?.genres,

    doubanVotes:   detail?.ratings_count,
    imdbRating:    omdbData?.imdbRating,              // ← OMDb 结果
    imdbVotes:     omdbData?.imdbVotes,               // ← OMDb 结果
```

```typescript
/** 从豆瓣 aka 中提取英文片名 */
function extractEnglishTitle(aka: string[], originalTitle: string): string | null {
  // 优先 original_title
  if (originalTitle && /^[\x00-\x7F\s]+$/.test(originalTitle) && /[a-zA-Z]/.test(originalTitle)) {
    return originalTitle
  }
  // 从 aka 中找纯英文/含 Unicode 罗马数字的
  for (const alias of aka) {
    if (/^[\x00-\x7F\sⅠⅡⅢⅣⅤⅥ\-'&!.,():]+$/.test(alias) && /[a-zA-Z]{3,}/.test(alias)) {
      return alias.trim()
    }
  }
  return null
}
```

### 9.3 直接推送实现 (电影 & 剧集)

```typescript
// Express route — 电影和剧集共用同一逻辑
app.get('/api/stream/movie/:id/direct', handleDirectStream('movie'))
app.get('/api/stream/episode/:id/direct', handleDirectStream('episode'))

function getMimeType(ext: string): string {
  const mimeTypes: Record<string, string> = {
    '.mp4': 'video/mp4',
    '.webm': 'video/webm',
    '.mkv': 'video/x-matroska',
    '.avi': 'video/x-msvideo',
    '.mov': 'video/quicktime',
    '.ts': 'video/mp2t',
    '.wmv': 'video/x-ms-wmv',
  }
  return mimeTypes[ext] || 'application/octet-stream'
}

function handleDirectStream(type: 'movie' | 'episode') {
  return (req, res) => {
    const record = type === 'movie'
      ? db.getMovie(req.params.id)
      : db.getEpisode(req.params.id)

    const stat = fs.statSync(record.file_path)
    const range = req.headers.range
    const mimeType = getMimeType(record.ext)

    if (range) {
      // 支持断点续传
      const parts = range.replace(/bytes=/, '').split('-')
      const start = parseInt(parts[0], 10)
      const end = parts[1] ? parseInt(parts[1], 10) : stat.size - 1
      res.writeHead(206, {
        'Content-Range': `bytes ${start}-${end}/${stat.size}`,
        'Accept-Ranges': 'bytes',
        'Content-Length': end - start + 1,
        'Content-Type': mimeType
      })
      fs.createReadStream(record.file_path, { start, end }).pipe(res)
    } else {
      res.writeHead(200, {
        'Content-Length': stat.size,
        'Content-Type': mimeType
      })
      fs.createReadStream(record.file_path).pipe(res)
    }
  }
}
```

> **为什么直接播放？** 现代浏览器原生支持多种视频格式（MKV 已获得广泛支持），直接播放延迟低、CPU 占用少，无需额外转码或容器转换。

## 附录 A: 问题与解决方案

### A.1 服务器崩溃问题

**问题描述**：服务器启动后立即崩溃，无法正常运行。

**原因分析**：`sqlite` 库与 Node.js v25.0.0 存在兼容性问题，导致异步数据库操作失败。

**解决方案**：将 `sqlite` 库迁移到 `better-sqlite3`，使用同步数据库操作替代异步操作。

**修改文件**：
- `server/db.ts` - 使用 `better-sqlite3` 初始化数据库
- `server/routes/movies.ts` - 移除 `await` 和 `finalize()` 调用
- `server/routes/stream.ts` - 移除 `finalize()` 调用
- `server/routes/genres.ts` - 添加类型转换
- `server/services/fileScanner.ts` - 转换为同步文件操作
- `server/services/metadataFetcher.ts` - 使用同步数据库方法

**关键代码变更**：
```typescript
// 旧代码 (sqlite)
const stmt = await db.prepare('SELECT * FROM movies')
const movies = await stmt.all()
await stmt.finalize()

// 新代码 (better-sqlite3)
const stmt = db.prepare('SELECT * FROM movies')
const movies = stmt.all()
// 无需 finalize()
```

### A.2 海报不显示问题

**问题描述**：刷新元数据后，海报仍然不显示，数据库中 `local_poster` 字段为 `null`。

**原因分析**：
1. 元数据获取逻辑中，海报下载失败但状态仍被标记为 `fetched`

**解决方案**：
1. 优化海报下载逻辑，确保海报成功下载后才更新数据库
2. 添加豆瓣图片的 Referer 请求头

**修改文件**：
- `server/services/metadataFetcher.ts` - 修复海报下载和保存逻辑，添加 Referer 请求头

### A.3 页面黑屏问题

**问题描述**：点击电影卡片进入详情页时黑屏，返回主页也是黑屏，需要刷新才能恢复。

**原因分析**：前端 `MovieDetailPage.tsx` 中尝试将逗号分隔的字符串解析为 JSON，导致 JavaScript 错误，页面渲染失败。

**解决方案**：修改解析逻辑，将逗号分隔的字符串正确转换为数组。

**修改文件**：
- `client/src/pages/MovieDetailPage.tsx`

**关键代码变更**：
```typescript
// 旧代码 (错误)
const cast = movie.cast ? JSON.parse(movie.cast) : []
const genres = movie.genres ? JSON.parse(movie.genres) : []

// 新代码 (正确)
const cast = movie.cast ? movie.cast.split(',').filter(Boolean) : []
const genres = movie.genres ? movie.genres.split(',').filter(Boolean) : []
```

### A.4 中文文件名乱码问题

**解决方案**：在文件扫描时进行正确的编码转换。

### A.6 mkv格式电影无法播放

**问题描述**：mkv扩展名的电影无法播放，但mp4格式可以正常播放。

**原因分析**：
1. 后端只允许 `.mp4` 和 `.webm` 格式直接播放
2. Content-Type 使用了错误的格式

**解决方案**：
1. 在 `constants.ts` 中添加 `.mkv`, `.avi`, `.mov` 到支持列表
2. 在 `stream.ts` 中添加 MIME 类型映射函数

**修改文件**：
- `shared/constants.ts` - 添加支持的视频格式
- `server/routes/stream.ts` - 添加 MIME 类型映射

**关键代码变更**：
```typescript
// constants.ts
export const SUPPORTED_VIDEO_EXTENSIONS = ['.mp4', '.webm', '.mkv', '.avi', '.mov', '.ogg']

// stream.ts
function getMimeType(ext: string): string {
  const mimeTypes: Record<string, string> = {
    '.mp4': 'video/mp4',
    '.webm': 'video/webm',
    '.mkv': 'video/x-matroska',
    '.avi': 'video/x-msvideo',
    '.mov': 'video/quicktime',
    '.ogg': 'video/ogg'
  }
  return mimeTypes[ext.toLowerCase()] || 'application/octet-stream'
}
```
